## 一、总览

### 1.1 目标

实现笔记备注功能，允许用户在不修改原文的基础上附加说明和讨论，形成"笔记→备注→回复"的数据链。

### 1.2 涉及文件

| 文件 | 改动类型 | 说明 |
|------|:--------:|------|
| `core/database.py` | 修改 | `_init_schema()` 新增 type/parent_id 字段+索引 |
| `core/models.py` | 修改 | `Note` 类新增 type/parent_id 字段 |
| `core/api_server.py` | 修改 | 新增 3 个端点 + 修改 3 个现有端点 |
| `core/rag_service.py` | 修改 | AI 问答上下文包含备注 |
| `web/src/lib/api.ts` | 修改 | 新增 annotation API 函数 |
| `web/src/pages/NoteEditor.tsx` | 修改 | 新增备注区块 UI |
| `web/src/pages/Notes.tsx` | 修改 | 列表默认过滤 type='note' |
| `web/src/pages/Settings.tsx` | 无需改动 | — |
| `web/src/index.css` | 可能修改 | 备注区块样式 |

## 二、Phase 计划

### Phase 1：后端数据层 + API（预估：1.5h）

**任务：**

- [ ] `database.py`: `_init_schema()` 中新增 type/parent_id 字段迁移 + 索引
- [ ] `models.py`: `Note` 类增加 `type` / `parent_id` / `annotation_count` 字段
- [ ] `api_server.py`: 新增 `AnnotationCreate` Pydantic 模型
- [ ] `api_server.py`: 新增 `GET /api/notes/{id}/annotations`（树形嵌套返回）
- [ ] `api_server.py`: 新增 `POST /api/notes/{id}/annotations`
- [ ] `api_server.py`: 新增 `DELETE /api/annotations/{id}`
- [ ] `api_server.py`: 修改 `GET /api/notes` 默认 `type='note'`
- [ ] `api_server.py`: 修改 `GET /api/notes/{id}` 返回 `annotation_count`

**验证方法：**
```bash
# 创建备注
curl -X POST /api/notes/1/annotations -d '{"content":"test"}'

# 列出备注
curl /api/notes/1/annotations

# 笔记列表不显示备注
curl /api/notes | jq 'length'  # 应与备注无关
```

### Phase 2：前端备注区块 UI（预估：2h）

**任务：**

- [ ] `api.ts`: 新增 `getAnnotations()` / `createAnnotation()` / `deleteAnnotation()`
- [ ] `NoteEditor.tsx`: 新增 `<AnnotationSection>` 组件（见详细设计 3.2 节）
- [ ] 备注列表展示（头像、内容、时间、回复缩进）
- [ ] 备注输入框（Enter 发送，Shift+Enter 换行）
- [ ] 回复模式（点击"回复"→自动填充 parent_id）
- [ ] 删除按钮（悬停显示，确认后删除）
- [ ] `index.css`: 备注区块专用样式（缩进线、头像、时间等）

**验证方法：**
- 打开笔记详情页 → 底部可见备注区
- 输入备注 → Enter → 备注出现在列表中
- 点"回复" → 输入框缩进 → 发送 → 嵌套显示

### Phase 3：搜索过滤 + 页面布局调整（预估：1h）

**任务：**

- [ ] `Notes.tsx`: 搜索/列表默认 `type='note'`
- [ ] `NoteEditor.tsx`: 页面布局调整为 标签→摘要→正文→备注→反向链接
- [ ] AI 摘要区块：短笔记隐藏（`<100字`），长笔记可折叠

**验证方法：**
- 笔记列表不应显示 type='annotation' 的记录
- 页面顺序正确
- 短笔记无摘要区，长笔记摘要区默认折叠

### Phase 4：AI 摘要持久化 + AI 标签建议模式（预估：1.5h）

**任务：**

- [ ] `database.py`: 新增 `ai_summary TEXT` 字段
- [ ] `api_server.py`: `POST /api/notes/{id}/summarize` 写入 `ai_summary` 字段而非仅返回
- [ ] `api_server.py`: `GET /api/notes/{id}` 返回 `ai_summary`
- [ ] `NoteEditor.tsx`: AI 标签改为建议模式（点标签→写入笔记）
- [ ] `api_server.py`: 新增/PUT端点：接受 AI 推荐标签

**验证方法：**
- 生成一次摘要 → 刷新页面 → 摘要不消失
- AI 推荐标签 → 点击单个标签 → 标签写入笔记

### Phase 5（未来）：行内批注 + 备注通知

- 选中文本→添加批注（需 `anchor_start` / `anchor_end` 字段）
- 新备注通知（WebSocket 推送）
- 备注编辑功能
