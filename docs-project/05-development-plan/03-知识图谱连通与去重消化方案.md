### 2.2 `SemanticService.scan_duplicates()`

```python
def scan_duplicates(
    self,
    workspace_id: Optional[int] = None,
    threshold: float = 0.6,
    limit: int = 50
) -> List[Dict]:
    """
    全库扫描重复笔记组
    
    策略: 遍历所有笔记 → 对每篇调 find_similar_notes() → 
          相似度 > threshold 的分到一组 → 合并重叠组
    
    Returns:
        [{'notes': [Note, Note], 'max_similarity': 0.87, 'common_tags': [...]}, ...]
    """
```

**算法思路**（union-find 分组）:
1. 获取所有笔记 ID 列表
2. 对每个笔记调 `find_similar_notes()`，收集相似度 > threshold 的对
3. 用并查集合并有交集的对 → 形成重复组
4. 每个组返回成员列表 + 最高相似度 + 共同标签

### 2.3 `KnowledgeGraphService.suggest_consolidation()`

```python
def suggest_consolidation(
    self,
    workspace_id: Optional[int] = None
) -> Dict:
    """
    基于社区检测和相似度边，建议知识整理操作
    
    Returns:
        {
            'topics': [{'name': 'FastAPI', 'note_count': 8, 'core_ids': [...], 'fragment_ids': [...]}],
            'stale_candidates': [{'note_id': 1, 'days_since_update': 90, 'has_incoming_links': False}],
            'merge_suggestions': [{'group': [id1, id2], 'reason': 'high_similarity', 'similarity': 0.85}]
        }
    """
```

**数据来源**:
- `build_graph()` → 社区检测 (`detect_communities`) → 主题聚类
- `_build_similarity_edges()` → Jaccard > 0.6 的边 → 合并建议
- `notes.updated_at` + `activity_log` → 陈旧检测
