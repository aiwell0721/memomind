## 一、数据模型变更

### 1.1 notes 表新增字段

```sql
ALTER TABLE notes ADD COLUMN type      TEXT NOT NULL DEFAULT 'note';
ALTER TABLE notes ADD COLUMN parent_id INTEGER REFERENCES notes(id);
CREATE INDEX IF NOT EXISTS idx_notes_type ON notes(type);
CREATE INDEX IF NOT EXISTS idx_notes_parent ON notes(parent_id);
```

### 1.2 type 枚举值

| type 值 | 含义 | 展示规则 |
|---------|------|---------|
| `note` | 普通笔记（默认） | 笔记列表、搜索结果均显示 |
| `annotation` | 用户备注 | 仅在其所属笔记详情页展示 |
| `summary` | AI 摘要（缓存） | 暂不使用，预留 |

### 1.3 备注记录示例

| id | title | content | type | parent_id | tags |
|----|-------|---------|------|-----------|------|
| 1 | 2024 行业报告 | 原始报告正文... | `note` | NULL | [导入, 行业] |
| 2 | — | 注意：数据源是 2023 年的 | `annotation` | 1 | [待确认] |
| 3 | — | 同意，建议补充新数据 | `annotation` | 2 | [] |

备注的 title 字段约定为 `NULL` 或空字符串，前端渲染时自动显示为"对《xxx》的备注"。

## 二、后端 API 设计

### 2.1 新增端点

#### `GET /api/notes/{note_id}/annotations` — 列出备注

```
Response 200:
[
  {
    "id": 2,
    "parent_id": null,
    "content": "注意：数据源是 2023 年的",
    "author": "张三",
    "created_at": "2026-07-19T10:00:00",
    "updated_at": "2026-07-19T10:00:00",
    "replies": [           ← 嵌套回复（parent_id 指向该备注）
      {
        "id": 3,
        "parent_id": 2,
        "content": "同意，建议补充新数据",
        ...
      }
    ]
  }
]
```

#### `POST /api/notes/{note_id}/annotations` — 创建备注

```
Request:
{
  "content": "注意：数据源是 2023 年的",
  "parent_id": null        ← null=顶级备注，非null=回复某条备注
}

Response 201:
{
  "id": 2,
  "type": "annotation",
  "note_id": 1,
  "parent_id": null,
  "content": "注意：数据源是 2023 年的",
  "created_at": "..."
}
```

**后端逻辑：**

```python
@app.post("/api/notes/{note_id}/annotations")
def create_annotation(note_id: int, req: AnnotationCreate, _: dict = Depends(verify_token)):
    # 1. 验证原笔记存在
    note = db.execute("SELECT id FROM notes WHERE id = ?", (note_id,)).fetchone()
    if not note:
        raise HTTPException(404, "笔记不存在")
    
    # 2. 创建备注笔记
    cursor = db.execute(
        """INSERT INTO notes (title, content, type, parent_id, tags)
           VALUES (?, ?, 'annotation', ?, ?)""",
        ("", req.content, req.parent_id or note_id, "[]")
    )
    annotation_id = cursor.lastrowid
    
    # 3. 返回
    return {"id": annotation_id, ...}
```

**关键设计决策：**
- `parent_id` 指向**目标笔记的 id**（顶级备注）或**父备注的 id**（回复）
- 为简化第一版，`parent_id` 统一存储：顶级备注存 `note_id`，回复存 `parent_annotation_id`
- 前端通过 `parent_id` 是否为 `note_id` 来判断层级

#### `DELETE /api/annotations/{annotation_id}` — 删除备注

```
Response 200: {"status": "deleted"}
```

- 仅创建者可删除
- 级联删除所有子回复

```

│  标题                            │

└──────────────────────────────────┘
```

### 3.2 备注区块组件

```typescript
// 组件树
NoteEditor
  └── AnnotationSection
        ├── AnnotationHeader  ("💬 备注 (3)")
        ├── AnnotationList
        │     ├── AnnotationItem  (顶级备注)
        │     │     ├── AnnotationContent
        │     │     ├── AnnotationActions  (回复/编辑/删除)
        │     │     └── AnnotationList  (嵌套回复)
        │     └── ...
        └── AnnotationInput  (输入框 + 发送按钮)
```

### 3.3 关键交互

| 操作 | 行为 |
|------|------|
| **创建备注** | 输入框 → Enter/发送 → `POST /api/notes/{id}/annotations` → 追加到列表 |
| **回复备注** | 点"回复" → 输入框聚焦，自动缩进 → 发送时带 `parent_id` |
| **删除备注** | 悬停显示删除按钮 → 确认 → `DELETE /api/annotations/{id}` |
| **编辑备注** | Phase 1 不做，Phase 2 加（点击备注内容直接编辑） |

### 3.4 样式要点

- 备注区块与正文有明显视觉分隔（灰线 + 缩进）
- 备注头部带用户头像（首字母）和创建时间
- 回复有左侧缩进线（thread 缩进风格）
- 输入框支持多行（Shift+Enter 换行，Enter 发送）

## 四、FTS 全文搜索备注

备注内容自动参与 FTS 索引，无需额外建表：

```python
# database.py 中的 _fts_rebuild_all() 已自动覆盖
# 因为备注也是 notes 表中的记录，INSERT/UPDATE/DELETE 时自动同步 FTS
```

但搜索时需要决定：搜索默认是否包含备注？

**决策：** 默认不包含（`type='note'`），用户可通过高级选项包含备注。
理由：笔记列表主要展示"笔记"，备注应是辅助信息。

## 五、AI 问答（RAG）中的备注

```python
# rag_service.py 中构建上下文时，增加备注内容
def _build_context(self, note_ids: list[int]) -> str:
    notes = self.db.execute(
        "SELECT content FROM notes WHERE id IN ({}) AND type = 'note'".format(...)
    )
    annotations = self.db.execute(
        "SELECT content FROM notes WHERE parent_id IN ({}) AND type = 'annotation'".format(...)
    )
    # 备注内容追加到上下文末尾，优先级高于原文
    context = "\n".join(note.content for note in notes)
    if annotations:
        context += "\n\n【备注信息】\n" + "\n".join(a.content for a in annotations)
    return context
```

**备注上下文权重：** 追加在原文之后，AI 自然会对后出现的内容给予更多关注，间接实现了"备注比原文更重要"的效果。
