## 2. LoCoMo 检索准确率

### 测试设计
- 30 条模拟中文对话记忆（覆盖项目、学习、个人、工作、会议、旅行 6 类）
- 50 个 Single-hop 事实检索问题
- 混合搜索（FTS5 + TF-IDF 语义），Top-5 命中即计为正确

### 结果

```
FTS5 基线:     80.0%  (40/50)
FTS5+TF-IDF:   82.0%  (41/50)
```

### 分析
- 关键词查询命中率高（"JWT token认证" → 命中 API 接口规范讨论）
- 9 个未命中集中在需要部分匹配的场景（如 "React TypeScript前端技术栈" vs 目标笔记中的 "React+TypeScript"）
- FTS5+jieba 对中文短关键词检索表现良好
- TF-IDF 语义搜索额外命中 1 题，提升有限但方向正确

---

## 3. 搜索延迟

### 测试设计
- 3 个数据规模：500 / 1000 / 2000 条笔记
- 10 个典型查询，每个 30 次迭代（含预热）

### 结果

| 规模 | P50 | P95 | P99 |
|------|-----|-----|-----|
| 500 | 1.42ms | 1.72ms | 2.63ms |
| 1000 | 2.08ms | 2.69ms | 2.80ms |
| 2000 | 3.30ms | 4.56ms | 5.20ms |

### 分析
- SQLite FTS5 + BM25 在当前数据规模下延迟极低
- 2000 条笔记 P95 仅 4.56ms，远低于 50ms 目标
- 延迟随数据量线性增长，可推测 10000 条笔记 P95 约 20ms

---

## 4. 中文搜索质量

### 测试设计
- 50 条中文笔记（覆盖编程、数据库、AI、前端、生活等 10 类）
- 50 个关键词查询（每查询预定义 1-3 条相关笔记）
- 度量精确率、召回率、F1

### 结果

```
FTS5 关键词搜索:  P=0.7%  R=2.0%  F1=1.0%
FTS5+TF-IDF 混合: P=2.0%  R=20.0% F1=3.6%
```

### 分析（待调查）
- FTS5 大部分查询返回语法错误（`fts5: syntax error near "."`）
- 推测根因：测试笔记内容含 FTS5 特殊字符（如 `docker-compose.yml`、`github.com`）导致索引异常
- LoCoMo 测试中 FTS5 表现正常（80%），说明问题出在**笔记内容而非查询流程**
- **待修复**：在 tokenizer 层过滤/转义 FTS5 特殊字符
