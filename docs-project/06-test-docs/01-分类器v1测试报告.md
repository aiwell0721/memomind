## 💡 改进建议

### 优先级 1: 提高置信度分数 (关键)

**问题**: 66% 的样本置信度在 0.50-0.69 区间，高置信度仅 6%

**原因分析**:
1. 历史准确率数据不足 - 当前仅 6 个位置有历史数据，大量位置使用默认值 0.7
2. 关键词匹配度计算过于严格 - 需要匹配 50% 关键词才能获得 1.0 分
3. 标题明确度评分标准偏高

**改进措施**:
```python
# 建议调整置信度权重
置信度 = 0.5 × 关键词匹配度 + 0.3 × 历史准确率 + 0.2 × 标题明确度
# 增加关键词匹配权重，因为这是最可靠的指标

# 优化关键词匹配计算
keyword_score = min(1.0, match_count / max(3, len(keywords)))  # 降低分母要求

# 增加历史数据覆盖
HISTORY_ACCURACY = {
    # ... 为所有 Room 添加默认历史数据
    "default": {"total": 5, "correct": 4}  # 默认 80% 准确率
}
```

### 优先级 2: 修复分类错误 (6 例)

**错误模式分析**:

| 错误类型 | 案例 | 原因 | 修复方案 |
|---------|------|------|----------|
| 性能关键词误判 | PR-008 笔记存储性能优化 → hot-cache | "性能"关键词在 hot-cache 权重过高 | 增加上下文判断，"存储 + 性能"应优先 note-storage |
| 链接关键词模糊 | PR-014 反向链接 → note-storage | "链接"在 note-storage 也有权重 | 增加"双向链接"、"编译"等专属关键词权重 |
| 日志关键词模糊 | PR-028 操作日志 → semantic-search | "记录"、"查询"关键词误匹配 | 增加"审计"、"日志"在 audit-system 的专属权重 |
| 监控关键词模糊 | PR-037 缓存监控 → audit-system | "监控"在 audit-system 权重高 | 增加"缓存命中率"作为 hot-cache 专属短语 |
| 跨模块集成 | PR-047 编译与缓存集成 | 多模块关键词冲突 | 实现多标签分类或主次要分类机制 |
| 结构关联 | PR-048 审计 + 宫殿 → palace-structure | "宫殿结构"关键词权重过高 | 降低通用结构词权重，增加业务关键词权重 |

**修复代码建议**:
```python
# 增加专属短语匹配（权重高于单个关键词）
ROOM_PHRASES = {
    "note-storage": ["笔记存储", "笔记 CRUD", "笔记管理"],
    "knowledge-compiler": ["双向链接", "知识编译", "Wiki 链接"],
    "hot-cache": ["缓存命中率", "LRU 缓存", "缓存预热"],
    "audit-system": ["操作日志", "审计日志", "健康度监控"],
}

# 短语匹配加分
for room, phrases in ROOM_PHRASES.items():
    if any(phrase in text for phrase in phrases):
        return room  # 直接返回，短语匹配优先级最高
```

### 优先级 3: 持续学习机制

1. **建立反馈循环**: 将人工修正的分类结果记录到 HISTORY_ACCURACY
2. **每周关键词回顾**: 分析低置信度 PR，手动分类后提取新关键词
3. **A/B 测试**: 对比不同权重配置的效果，选择最优参数
