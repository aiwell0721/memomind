# 核心功能详细说明

> 最后更新：2026-05-01

---

## 1. 笔记管理

### 基本操作

| 操作 | 说明 |
|------|------|
| 创建笔记 | 标题 + 内容（支持 Markdown），自动记录创建时间 |
| 编辑笔记 | 实时更新，自动触发版本历史和全文索引更新 |
| 删除笔记 | 软删除（可恢复），自动清理 FTS5 索引 |
| 列表查询 | 支持分页、按工作区过滤、按标签过滤 |

### 数据结构

```python
Note {
    id: int
    title: str
    content: str  # Markdown 格式
    tags: List[str]
    created_at: datetime
    updated_at: datetime
}
```

## 2. 全文搜索

基于 SQLite FTS5 引擎，BM25 相关度排序。

### 功能

| 功能 | 说明 |
|------|------|
| 关键词搜索 | 在标题、内容、标签中搜索 |
| 中文分词 | jieba 预处理，解决 FTS5 无法识别中文词边界的问题 |
| BM25 排序 | 相关度从高到低排序 |
| 关键词高亮 | 返回匹配的关键词片段 |
| 标签过滤 | 搜索时可指定标签 |
| 分页支持 | LIMIT/OFFSET 分页 |
| 自动同步 | 笔记增删改通过触发器自动更新 FTS5 索引 |

### 中文分词方案

写入时预处理：`jieba.cut()` → 空格分隔 → 写入 FTS5  
查询时同样分词 → 构建 FTS5 MATCH 查询

```
原始文本: "笔记软件的设计非常优雅"
分词结果: "笔记 软件 的 设计 非常 优雅"
```

## 3. 语义搜索

基于 TF-IDF + 余弦相似度的语义理解搜索。

### 功能

| 功能 | 说明 |
|------|------|
| TF-IDF 向量化 | 对笔记库构建词频-逆文档频率向量 |
| 余弦相似度 | 计算查询与每条笔记的语义相似度 |
| 混合搜索 | FTS5 关键词 + 语义相似度融合排序 |
| 自动清理缓存 | TF-IDF 模型可重建 |

### 混合搜索模式

```
最终得分 = α × FTS5_BM25得分 + (1-α) × 语义相似度得分
```

## 4. 标签系统

### 功能

| 功能 | 说明 |
|------|------|
| 创建标签 | 支持指定父标签（层级结构） |
| 标签别名 | 将重复标签合并到主标签 |

| 标签合并 | 合并两个标签及其关联笔记 |

### 层级示例

```
技术
├── Python
│   ├── FastAPI
│   └── SQLite
└── JavaScript
    └── React
```

## 5. 双向链接

使用 Wiki 语法 `[[笔记标题]]` 在笔记间建立链接。

### 功能

| 功能 | 说明 |
|------|------|
| 创建链接 | 在笔记内容中使用 `[[目标笔记]]` 语法 |
| 反向链接 | 查询"哪些笔记链接了我" |
| 链接统计 | 每条笔记的入链/出链数量 |
| 孤立笔记检测 | 找出没有链接关系的笔记 |
| 链接别名 | `[[目标笔记|显示文本]]` |

## 6. 版本历史

### 功能

| 功能 | 说明 |
|------|------|
| 自动保存版本 | 每次编辑自动创建新版本 |
| 版本查询 | 列出笔记的所有历史版本 |
| 版本恢复 | 恢复到指定历史版本 |
| 差异对比 | 两个版本间的标题/内容差异（统一 diff 格式） |
| 版本标签 | 给重要版本打标签 |
| 自动清理 | 按策略清理旧版本 |

### 版本数据结构

```python
Version {
    id: int
    note_id: int
    version_number: int
    title: str
    content: str
    tags: List[str]
    created_at: datetime
    version_tag: str  # 可选
}
```

## 16. 实时协作（WebSocket）

### 阶段一：简单广播 + 冲突检测（Phase 4）

| 功能 | 说明 |
|------|------|
| WebSocket 端点 | `ws://localhost:8000/ws/notes/{note_id}` |
| 房间管理 | 每笔记一个房间，同房间用户实时广播 |
| 内容同步 | 编辑操作广播给同房间所有用户 |
| 在线用户 | 显示谁正在编辑同一笔记 |
| 冲突检测 | 复用 conflict_service，并发编辑时记录冲突 |
| 心跳保活 | 30s 心跳，断开自动清理 |

### 阶段二：CRDT 冲突解决（后续版本）

| 功能 | 说明 |
|------|------|
| Yjs CRDT | 引入 Yjs 算法，自动解决并发冲突 |
| 操作转换 | 字符级操作，无冲突合并 |
| 前端集成 | 编辑器直接对接 CRDT |
