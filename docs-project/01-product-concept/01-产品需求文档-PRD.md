## 1. 背景与目标

### 1.1 业务背景

随着AI Agent技术的快速发展，知识工作者面临新的挑战和机遇：
- **信息过载**：传统笔记工具无法有效利用AI能力处理海量信息
- **上下文断裂**：AI助手缺乏长期记忆，每次对话都是新的开始
- **协作困境**：人类与AI、AI与AI之间缺乏有效的协作机制
- **数据锁定**：现有云笔记工具造成数据孤岛和迁移困难

### 1.2 用户痛点

| 用户类型 | 核心痛点 | 现有解决方案缺陷 |
|----------|----------|------------------|
| 开发者 | 代码片段、文档、笔记分散在不同工具 | Cursor只管代码，Notion不够技术 |
| 知识工作者 | 难以从海量笔记中快速找到需要的信息 | 传统搜索依赖关键词，缺乏语义理解 |
| AI早期采用者 | 希望构建个性化的AI助手但缺乏基础设施 | LangChain太复杂，Mem不够开放 |
| 团队负责人 | 团队知识难以沉淀和传承 | 协作工具缺乏AI能力，信息孤岛 |

### 1.3 产品目标

**愿景**：成为AI Agent时代的"记忆操作系统"

**目标**：
1. 为AI Agent提供持久化、结构化的记忆基础设施
2. 构建人机共享的知识协作空间
3. 支持多Agent协作的开放生态

**成功指标**：
| 指标 | 6个月 | 12个月 | 24个月 |
|------|-------|--------|--------|
| 注册用户 | 10,000 | 100,000 | 1,000,000 |
| 月活跃用户 | 3,000 | 30,000 | 300,000 |
| 付费转化率 | 5% | 8% | 12% |
| NPS评分 | >40 | >50 | >60 |

## 2. 用户故事

### 2.1 开发者用户

**故事1：代码知识管理**
> 作为一名全栈开发者，我希望将代码片段、技术文档和学习笔记统一管理，以便AI助手能够理解我的技术栈并提供个性化建议。

**故事2：AI Agent开发**
> 作为一名AI应用开发者，我希望有一个标准化的记忆接口，让我的Agent能够持久化存储和检索信息，而不是每次重启都丢失上下文。

### 2.2 知识工作者

**故事3：智能信息捕获**
> 作为一名产品经理，我希望通过语音或截图快速记录想法，AI自动分类、打标签并关联相关知识，而不是手动整理。

**故事4：语义搜索**
> 作为一名研究员，我希望用自然语言提问就能找到半年前的相关笔记，而不是记住具体关键词。

### 2.3 团队用户

**故事5：团队知识库**
> 作为一个10人创业团队，我们希望构建共享的知识空间，AI能够回答新员工的基础问题，减少重复培训。

**故事6：多Agent协作**
> 作为一个AI自动化团队，我们希望多个专业Agent（研究、写作、编程）能够共享记忆，协作完成复杂任务。

## 4. 业务流程

### 4.1 核心用户旅程

```
[发现] → [安装] → [创建笔记] → [AI辅助] → [搜索检索] → [分享协作]
  ↓         ↓          ↓           ↓           ↓           ↓
官网    下载      编辑器      AI建议      语义搜索    链接分享
社区    初始化    导入        自动标签    问答        协作邀请
```

### 4.2 笔记创建流程

```
用户创建笔记
    ↓
选择类型（文本/代码/剪藏/语音）
    ↓
编辑器打开
    ↓
AI实时分析 → 标签推荐 → 关联建议
    ↓
用户编辑内容
    ↓
保存 → 向量化 → 索引更新
    ↓
触发自动化（如设置规则）
```

### 4.3 AI搜索流程

```
用户输入查询
    ↓
查询理解（意图识别）
    ↓
向量检索 + 关键词检索
    ↓
结果重排序（相关性+时效性+用户偏好）
    ↓
生成答案（RAG）
    ↓
展示结果 + 引用来源
    ↓
用户反馈（改进模型）
```

### 4.4 API调用流程

```
第三方应用
    ↓
API认证（Token）
    ↓
权限检查
    ↓
请求处理
    ↓
数据操作（CRUD/Search）
    ↓
返回结果（JSON）
    ↓
Webhook通知（如配置）
```

#### 5.1.3 字段定义

```typescript
interface Note {
  id: string;                    // UUID
  title: string;                 // 标题，最大200字符
  content: string;               // Markdown内容
  contentType: 'markdown' | 'code' | 'canvas';
  folderId: string | null;       // 所属文件夹
  tags: string[];                // 标签数组
  createdAt: DateTime;           // 创建时间
  updatedAt: DateTime;           // 更新时间
  authorId: string;              // 作者ID
  version: number;               // 版本号
  metadata: {
    wordCount: number;
    readingTime: number;
    source?: string;             // 剪藏来源
    aiGenerated: boolean;        // 是否AI生成
  };
  embeddings?: Vector[];         // 向量表示（内部使用）
}
```

| 异常场景 | 处理方式 | 用户提示 |

### 5.4 API系统

#### 5.4.1 认证方式

**API Key认证**：
```http
Authorization: Bearer {api_key}
```

**OAuth 2.0**（Phase 2）：
- 支持第三方应用授权
- 细粒度权限控制

#### 5.4.2 核心API端点

| 端点 | 方法 | 描述 | 权限 |
|------|------|------|------|
| `/api/v1/notes` | GET | 获取笔记列表 | read |
| `/api/v1/notes` | POST | 创建笔记 | write |
| `/api/v1/notes/{id}` | GET | 获取笔记详情 | read |
| `/api/v1/notes/{id}` | PUT | 更新笔记 | write |
| `/api/v1/notes/{id}` | DELETE | 删除笔记 | write |
| `/api/v1/search` | POST | 语义搜索 | read |
| `/api/v1/ask` | POST | AI问答 | read |
| `/api/v1/folders` | GET | 获取文件夹 | read |
| `/api/v1/tags` | GET | 获取标签 | read |

#### 5.4.3 请求示例

**创建笔记**：
```json
POST /api/v1/notes
Content-Type: application/json
Authorization: Bearer sk_live_xxx

{
  "title": "Agent架构设计",
  "content": "## 核心概念\n\nAgent是...",
  "folderId": "folder_123",
  "tags": ["AI", "架构"]
}
```

**语义搜索**：
```json
POST /api/v1/search
Content-Type: application/json

{
  "query": "多Agent如何协作",
  "type": "semantic",
  "filters": {
    "tags": ["AI"]
  },
  "limit": 10
}
```

**AI问答**：
```json
POST /api/v1/ask
Content-Type: application/json

{
  "question": "总结我的Agent相关笔记",
  "context": {
    "folderId": "folder_123",
    "dateRange": "last_month"
  }
}
```

#### 5.4.4 响应格式

```json
{
  "success": true,
  "data": { ... },
  "meta": {
    "requestId": "req_xxx",
    "timestamp": "2026-04-12T10:00:00Z"
  }
}
```
