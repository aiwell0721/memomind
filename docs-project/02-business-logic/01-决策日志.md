### DEC-001: 产品定位调整

| 属性 | 内容 |
|------|------|
| **决策ID** | DEC-001 |
| **决策日期** | 2026-04-12 |
| **决策人** | 产品助理 |
| **决策主题** | 重新定义产品定位 |
| **状态** | ✅ 已决策 |

**决策背景**:
- 原定位"AI Agent记忆操作系统"过于宽泛
- 试图同时服务个人用户、AI Agent开发者、AI应用构建者
- 导致产品失去焦点，功能臃肿

**决策选项**:

| 选项 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A | 保持原定位 | 覆盖面广 | 失去焦点，难以建立优势 |
| B | 聚焦开发者 | 细分市场明确 | 市场规模较小 |
| C | 聚焦知识工作者 | 市场较大 | 竞争激烈 |
| **D** | **技术工作者（开发者+写作者）** | **市场适中，竞争差异化** | **需要明确定义** |

**决策结果**: 选择选项D

**新定位**: "为技术工作者打造的AI原生笔记工具"

**决策依据**:
1. 技术工作者对AI接受度高
2. 现有工具（Obsidian太复杂，Notion太笨重）存在痛点
3. 开发者群体有付费意愿
4. 可以逐步扩展到更广泛用户

**影响范围**:
- ✅ 用户画像重新定义
- ✅ 功能优先级调整
- ✅ 营销信息更新
- ✅ 技术栈选择

**相关文档**:
- PRD v2: `output/prd/memomind-prd-v2.md`

### DEC-002: MVP范围削减

| 属性 | 内容 |
|------|------|
| **决策ID** | DEC-002 |
| **决策日期** | 2026-04-12 |
| **决策人** | 产品助理 |
| **决策主题** | 大幅削减MVP功能范围 |
| **状态** | ✅ 已决策 |

**决策背景**:
- 原MVP包含50+功能项
- 8周开发周期完全不现实
- 需要70-80天工时，远超40工作日

**决策选项**:

| 选项 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A | 保持原范围，延长周期 | 功能完整 | 延期风险高 |
| B | 保持原范围，增加人力 | 按时完成 | 成本高，协调难 |
| **C** | **大幅削减功能** | **聚焦核心，降低风险** | **功能较少** |
| D | 分阶段发布 | 逐步验证 | 用户体验不连贯 |

**决策结果**: 选择选项C

**削减方案**:

**保留核心功能**:
1. Markdown笔记编辑器（基础）
2. 本地文件存储
3. 基础AI搜索（语义）
4. 简单AI问答
5. 跨平台桌面端

**删除/延后**:
- CLI → Phase 2
- 插件系统 → Phase 2
- 云同步 → Phase 2
- 多Agent协作 → Phase 3
- 本地LLM → 放弃
- 知识图谱 → Phase 2

**决策依据**:
1. 遵循MVP原则，最小可行产品
2. 降低开发风险
3. 更快验证核心价值
4. 根据反馈迭代

**影响范围**:
- ✅ PRD v2更新
- ✅ 开发计划调整
- ✅ 资源重新分配

### DEC-003: 技术栈简化

| 属性 | 内容 |
|------|------|
| **决策ID** | DEC-003 |
| **决策日期** | 2026-04-12 |
| **决策人** | 产品助理 |
| **决策主题** | 简化技术栈选择 |
| **状态** | ✅ 已决策 |

**决策背景**:
- 原技术栈：Tauri(Rust) + React + SQLite + LanceDB + Tantivy + ChromaDB
- 过于复杂，开发难度大
- Rust学习曲线陡峭

**决策选项**:

| 选项 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A | 保持原技术栈 | 性能更好 | 开发难度大 |
| **B** | **Electron + SQLite + OpenAI** | **成熟稳定，开发快** | **包体积较大** |
| C | Tauri + 简化后端 | 性能较好 | 仍需学习Rust |

**决策结果**: 选择选项B

**新技术栈**:
- 桌面端: Electron
- 前端: React + TypeScript
- 状态管理: Zustand
- 编辑器: CodeMirror 6
- 存储: SQLite（单一数据库）
- AI: OpenAI API
- 搜索: SQLite FTS + 向量相似度

**决策依据**:
1. Electron生态成熟，文档丰富
2. 团队熟悉技术栈
3. 开发速度快，风险低
4. MVP阶段性能不是首要考虑

**影响范围**:
- ✅ 架构文档更新
- ✅ 技术准备调整
- ✅ 招聘需求变更

### DEC-006: 商业模式确定

| 属性 | 内容 |
|------|------|
| **决策ID** | DEC-006 |
| **决策日期** | 2026-04-12 |
| **决策人** | 产品助理 |
| **决策主题** | 确定商业模式和定价策略 |
| **状态** | ✅ 已决策 |

**决策背景**:
- 原定价策略多个档位，不清晰
- AI API成本高昂
- 需要明确盈利路径

**决策选项**:

| 选项 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A | 完全免费，开源捐赠 | 用户增长快 | 收入不稳定 |
| B | 纯订阅制 | 收入可预测 | 用户门槛高 |
| **C** | **Freemium（免费+Pro订阅）** | **平衡增长和收入** | **需要设计转化路径** |
| D | 一次性购买 | 简单明了 | 无持续收入 |

**决策结果**: 选择选项C

**定价策略**:
- **Free**: 本地使用、基础AI（自带API Key）
- **Pro**: $8/月 或 $80/年
  - 云同步
  - 内置AI额度（1000次/月）
  - 优先支持

**决策依据**:
1. 免费降低使用门槛
2. Pro版覆盖成本并盈利
3. 允许用户自带API Key降低成本
4. 与竞品（Notion $8-10/月）对齐

**影响范围**:
- ✅ 商业模式画布
- ✅ 财务预测
- ✅ 产品功能规划
