**决策背景**:

- 试图同时服务个人用户、AI Agent开发者、AI应用构建者

2. 报告的修复点 + 用户优化建议清单已足够定位需要做什么；
3. TDD 路径下，先写失败测试再修复，本身保证不会重复"未回写"的隐患。

**实际修复落地（4 个独立 commit）**:

1. **修复: 补 mcp_server/`__main__`.py 支持 `python -m` 启动**  
   原因：报告提到的 ImportError 实际是 `python -m mcp_server` 找不到 `__main__`。`api/client.py` 的相对导入虽然写法可商榷，但 `python -c "from mcp_server import main; main()"` 能跑——按法则三"能跑就别动"，**未改 client.py 的 11 处相对导入**。
2. **修复: 中文全文搜索 0 命中（FTS5 + jieba 分词）**  
   - `core/database.py`：移除 SQLite 触发器，改用正则识别 `notes` 主表写操作并自动 jieba 分词刷新 FTS5；正则 `\bnotes\b(?!_)` 排除 `notes_fts`/`note_versions`/`note_tags`/`note_links`。
   - `core/search_service.py`：移除错误的 `'\\\\'` 转义（FTS5 不支持反斜杠）。
   - `core/api_server.py`：启动时 `reindex_notes()` 回填历史数据。
   - `tests/test_search.py`：新增 5 条"无人工分词"的真实场景 RED 测试。
3. **修复: Web Settings 创建用户缺 password 参数**  
   不采用清单建议的 `'default123'` 硬编码弱口令；改用 `window.prompt` 让管理员输入初始密码（≥6 位），未填或不足则中止。
4. **未修：WAL 文件残留（报告问题 4）**  
   这是部署流程问题，不是代码问题。后续在 `docs/部署指南` 中明确"删 db 必须同时删 db-wal / db-shm"即可，无需改代码。
