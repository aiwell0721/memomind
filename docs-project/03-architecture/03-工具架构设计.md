### 1.2 架构设计原则

| 原则 | 说明 | 实现方式 |
|------|------|---------|
| **本地优先** | 数据首先存储在本地，用户完全掌控 | SQLite + 本地文件系统 |
| **AI First** | 架构设计考虑 AI Agent 的需求 | 向量数据库 + 记忆服务 |
| **开放集成** | 所有功能通过 API 暴露 | RESTful API + WebSocket |
| **模块化** | 服务间松耦合，可独立扩展 | 微服务架构 |
| **可扩展** | 支持插件扩展机制 | 插件服务 + SDK |

## 2. 技术栈选择

### 2.1 前端技术栈

| 组件 | 技术选型 | 选择理由 |
|------|---------|---------|
| **框架** | React 18 + TypeScript | 生态成熟，类型安全 |
| **桌面框架** | Tauri (Rust) | 轻量、安全、性能好 |

| **构建工具** | Vite | 快速、配置简单 |

### 2.2 后端技术栈

| 组件 | 技术选型 | 选择理由 |
|------|---------|---------|
| **运行时** | Node.js 20+ / Deno | 生态丰富、适合 I/O 密集型 |

| **API 协议** | REST + WebSocket | 通用、实时性好 |

### 2.3 数据存储技术栈

| 组件 | 技术选型 | 选择理由 |

### 3.2 数据库 Schema

#### 3.2.1 用户与权限

```sql
-- 用户表
CREATE TABLE users (
    id TEXT PRIMARY KEY,
    username TEXT UNIQUE NOT NULL,
    email TEXT UNIQUE,
    password_hash TEXT,
    api_key TEXT UNIQUE,
    settings JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 工作空间表
CREATE TABLE workspaces (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    name TEXT NOT NULL,
    path TEXT NOT NULL,
    settings JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- Agent 表
CREATE TABLE agents (
    id TEXT PRIMARY KEY,
    workspace_id TEXT NOT NULL,
    name TEXT NOT NULL,
    description TEXT,
    type TEXT DEFAULT 'custom',
    config JSON,
    api_key TEXT UNIQUE,
    permissions JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE
);
```

#### 3.2.2 笔记系统

```sql
-- 笔记表
CREATE TABLE notes (
    id TEXT PRIMARY KEY,
    workspace_id TEXT NOT NULL,
    title TEXT NOT NULL,
    content TEXT,
    path TEXT NOT NULL,
    hash TEXT,
    size INTEGER,
    is_template BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE
);

-- 标签表
CREATE TABLE tags (
    id TEXT PRIMARY KEY,
    workspace_id TEXT NOT NULL,
    name TEXT NOT NULL,
    color TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(workspace_id, name),
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE
);

-- 笔记标签关联表
CREATE TABLE note_tags (
    note_id TEXT NOT NULL,
    tag_id TEXT NOT NULL,
    PRIMARY KEY (note_id, tag_id),
    FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);

-- 链接表（双向链接）
CREATE TABLE links (
    id TEXT PRIMARY KEY,
    source_note_id TEXT NOT NULL,
    target_note_id TEXT,
    target_title TEXT,
    link_type TEXT DEFAULT 'wiki',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_note_id) REFERENCES notes(id) ON DELETE CASCADE,
    FOREIGN KEY (target_note_id) REFERENCES notes(id) ON DELETE SET NULL
);
```

#### 3.2.3 记忆系统

```sql
-- 会话表
CREATE TABLE sessions (
    id TEXT PRIMARY KEY,
    agent_id TEXT NOT NULL,
    status TEXT DEFAULT 'active',
    context JSON,
    token_count INTEGER DEFAULT 0,
    expires_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (agent_id) REFERENCES agents(id) ON DELETE CASCADE

-- 记忆表
CREATE TABLE memories (
    id TEXT PRIMARY KEY,
    session_id TEXT,
    agent_id TEXT NOT NULL,
    content TEXT NOT NULL,
    type TEXT DEFAULT 'fact',

    FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE SET NULL,
    FOREIGN KEY (agent_id) REFERENCES agents(id) ON DELETE CASCADE

-- 记忆关联表
CREATE TABLE memory_links (
    id TEXT PRIMARY KEY,
    source_memory_id TEXT NOT NULL,
    target_memory_id TEXT NOT NULL,
    relation_type TEXT DEFAULT 'related',
    strength REAL DEFAULT 0.5,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_memory_id) REFERENCES memories(id) ON DELETE CASCADE,
    FOREIGN KEY (target_memory_id) REFERENCES memories(id) ON DELETE CASCADE

```

## 4. API 设计规范

### 4.1 REST API 设计原则

- **资源导向**：URL 表示资源，HTTP 方法表示操作
- **版本控制**：URL 中包含版本号 `/api/v1/`
- **标准方法**：GET/POST/PUT/PATCH/DELETE
- **标准响应**：统一响应格式，包含 code/message/data
- **分页**：列表接口支持分页 `?page=1&limit=20`
- **过滤**：支持查询参数过滤 `?type=fact&importance=gte:0.8`

### 4.2 标准响应格式

```typescript
// 成功响应
interface SuccessResponse<T> {
  code: 0;
  message: "success";
  data: T;
  meta?: {
    page: number;
    limit: number;
    total: number;
  };
}

// 错误响应
interface ErrorResponse {
  code: number;        // 非零错误码
  message: string;     // 错误信息
  details?: any;       // 详细错误信息
}
```

```
  │                          │                            │                      │
```

### 5.2 安全措施

| 层面 | 措施 | 实现 |
|------|------|------|
| **传输安全** | HTTPS/TLS | 强制 HTTPS |
| **认证** | JWT + API Key | HS256 签名 |
| **授权** | RBAC | 角色权限控制 |
| **输入验证** | 参数校验 | Joi/Zod |
| **速率限制** | 请求限流 | Redis 计数 |
| **数据加密** | 敏感字段加密 | AES-256 |
| **审计日志** | 操作记录 | 数据库日志 |

### 5.3 安全策略

