## 1. 架构概述

### 1.1 设计原则

1. **本地优先**：用户数据首先存储在本地，确保数据主权和隐私
2. **开放格式**：使用开放标准（Markdown + JSON），避免数据锁定
3. **AI原生**：从底层架构支持AI能力，而非简单附加
4. **模块化**：插件化架构，支持功能扩展和定制
5. **跨平台**：一套代码支持多平台（桌面端优先，移动端后续）

### 1.2 技术栈选型

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **桌面端** | Tauri (Rust + Web) | 轻量、安全、跨平台 |
| **前端** | React + TypeScript | 生态丰富、类型安全 |
| **状态管理** | Zustand + TanStack Query | 轻量、高效 |
| **编辑器** | Monaco Editor / CodeMirror 6 | 功能强大、可扩展 |
| **后端服务** | Rust (Axum) / Node.js | 性能 + 生态平衡 |
| **数据库** | SQLite (本地) + PostgreSQL (云) | 轻量本地 + 强大云端 |
| **向量数据库** | LanceDB / Chroma (本地) | 本地优先的向量存储 |
| **搜索** | Tantivy (本地) + Meilisearch (云) | 高性能全文搜索 |
| **AI/ML** | ONNX Runtime / llama.cpp | 本地AI推理 |
| **同步** | CRDT (Yjs) / 自定义协议 | 实时协作基础 |

#### 2.2.3 混合搜索策略

```typescript
async function hybridSearch(query: string, options: SearchOptions): Promise<SearchResult[]> {
  // 1. 并行执行向量和关键词搜索
  const [vectorResults, keywordResults] = await Promise.all([
    vectorSearch.search(query, { limit: 50 }),
    fullTextSearch.search(query, { limit: 50 })
  ]);
  
  // 2. 融合结果（RRF - Reciprocal Rank Fusion）
  const fusedResults = reciprocalRankFusion(vectorResults, keywordResults, {
    k: 60,  // RRF常数
    weights: { vector: 0.6, keyword: 0.4 }
  });
  
  // 3. 重排序（可选，使用轻量级模型）
  const rerankedResults = await rerank(fusedResults, query);
  
  // 4. 返回最终结果
  return rerankedResults.slice(0, options.limit || 20);
}
```

## 3. 数据流设计

### 3.1 笔记创建流程

```
用户操作
    ↓
[UI] 编辑器组件
    ↓
[Core] NoteService.create()
    ├─→ 验证输入数据
    ├─→ 生成ID和时间戳
    ├─→ 写入文件系统 (Markdown)
    ├─→ 写入元数据数据库 (SQLite)
    ├─→ 异步：生成向量嵌入
    ├─→ 异步：更新搜索索引
    ├─→ 异步：AI标签推荐（如启用）
    └─→ 触发事件：note.created
    ↓
[Event] 通知相关服务
    ├─→ 同步服务：加入待同步队列
    ├─→ 插件系统：广播事件
    └─→ UI：更新笔记列表
```

### 3.2 搜索流程

```
用户输入查询
    ↓
[UI] 搜索组件
    ↓
[Core] SearchService.search()
    ├─→ 解析查询（自然语言/关键词）
    ├─→ 并行执行：
    │   ├─→ 向量搜索 (LanceDB)
    │   └─→ 关键词搜索 (Tantivy)
    ├─→ 融合结果 (RRF算法)
    ├─→ 过滤和排序
    ├─→ 生成高亮片段
    └─→ 返回结果
    ↓
[UI] 显示搜索结果
    ├─→ 结果列表
    ├─→ 高亮匹配内容
    └─→ 支持进一步筛选
```

1. **本地优先**：敏感数据默认本地处理
2. **可选云端**：AI功能可选择本地模型
3. **数据最小化**：只发送必要数据到云端
4. **透明控制**：用户清楚知道数据去向
5. **随时导出**：支持完整数据导出
