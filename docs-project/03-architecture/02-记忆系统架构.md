**文档版本**: v1.1

## 1. 概述

### 1.1 设计目标

构建一个面向AI Agent的持久化记忆系统，实现：
1. **长期记忆存储**：跨会话保持上下文
2. **高效检索**：毫秒级的语义搜索
3. **分层管理**：工作记忆、短期记忆、长期记忆
4. **多Agent共享**：支持Agent间记忆协作
5. **人机融合**：人类笔记与AI记忆统一

### 1.2 核心概念

| 概念 | 定义 | 示例 |
|------|------|------|
| **记忆单元** | 最小存储单位，对应一条笔记/片段 | 一个概念、一段对话 |
| **记忆类型** | 按内容和用途分类 | 事实、程序、情景、语义 |
| **记忆层级** | 按时效性分层 | 工作/短期/长期记忆 |
| **记忆关联** | 记忆间的连接关系 | 相关、因果、层次 |
| **记忆权重** | 重要性和访问频率 | 0-1之间的浮点数 |

### 2.1 记忆类型分类

```

├── 陈述性记忆 (Declarative)
│   ├── 语义记忆 (Semantic)      - 概念、事实
│   │   ├── 领域知识
│   │   ├── 实体信息
│   │   └── 概念关系

│       ├── 对话历史

│       └── 决策过程
├── 程序性记忆 (Procedural)

│   ├── 偏好记忆                 - 用户偏好

└── 元记忆 (Meta-memory)

    ├── 置信度记忆               - 可靠性评估
    └── 访问记忆                 - 使用统计
```

### 3.2 数据流

```
创建/更新记忆
    ↓
[1] 写入工作记忆 (内存)
    ↓
[2] 异步持久化到短期记忆 (SQLite)
    ├─→ 生成向量嵌入
    ├─→ 更新搜索索引
    └─→ 触发关联分析
    ↓
[3] 重要性评估
    ├─→ 高重要性 → 提升到长期记忆
    └─→ 低重要性 → 保留在短期记忆
    ↓
[4] 定期归档 (冷数据迁移)
    ├─→ 压缩存储
    └─→ 云端备份
```

### 6.2 遗忘算法

```typescript
interface ForgettingAlgorithm {
  // 计算记忆留存值
  calculateRetention(memory: MemoryUnit): number;
  
  // 决定是否需要遗忘
  shouldForget(memory: MemoryUnit): boolean;
  
  // 执行遗忘
  forget(memoryId: string, strategy: ForgetStrategy): Promise<void>;
}

// 基于艾宾浩斯遗忘曲线的实现
function calculateRetention(memory: MemoryUnit): number {
  const now = Date.now();
  const age = now - memory.metadata.createdAt.getTime();
  const accesses = memory.metadata.accessCount;
  const importance = memory.metadata.importance;
  
  // 基础遗忘曲线
  const baseRetention = Math.exp(-age / (86400000 * 30)); // 30天半衰期
  
  // 访问强化
  const accessBoost = Math.min(accesses * 0.1, 0.5);
  
  // 重要性保护
  const importanceBoost = importance * 0.3;
  
  return Math.min(baseRetention + accessBoost + importanceBoost, 1);
}
```
